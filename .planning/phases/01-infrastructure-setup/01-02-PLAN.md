<task type="auto">
  <name>Task 2: Create Docker Compose Setup</name>
  <files>docker-compose.yml</files>
  <read_first>
    - docker-compose.yml
    - livekit.yaml
    - Caddyfile
  </read_first>
  <action>
    Create `docker-compose.yml` with two services: `livekit` and `caddy`.
    For `livekit`:
    - Use image `livekit/livekit-server:latest`.
    - Map ports directly on the host (per D-03): `7880:7880`, `7881:7881`, `3478:3478/udp`, `5349:5349/tcp`, `50000-60000:50000-60000/udp`. Check `user-livekit-docs` MCP if `network_mode: "host"` is strictly required or if port mapping is sufficient for WebRTC UDP.
    - Mount `./livekit.yaml:/livekit.yaml`.
    - Apply `env_file: .env`.
    For `caddy`:
    - Use image `caddy:latest`.
    - Map ports `80:80`, `443:443`.
    - Mount `./Caddyfile:/etc/caddy/Caddyfile`, `caddy_data:/data`, `caddy_config:/config`.
    - Apply `env_file: .env`.
    Add named volumes for `caddy_data` and `caddy_config`.
  </action>
  <verify>
    <automated>grep -q "livekit/livekit-server" docker-compose.yml && grep -q "caddy:" docker-compose.yml</automated>
  </verify>
  <acceptance_criteria>
    - `docker-compose.yml` contains service `livekit` with image `livekit/livekit-server`
    - `docker-compose.yml` contains service `caddy` with image `caddy`
    - `docker-compose.yml` maps WebRTC UDP ports on `livekit` service
    - `docker-compose.yml` mounts `./livekit.yaml` and `./Caddyfile`
  </acceptance_criteria>
  <done>Docker compose configuration orchestrates LiveKit and Caddy securely.</done>
</task>
